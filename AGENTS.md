# Never use --no-verify.

# Branch names and PR titles should be under 100 chars

# Branch name conventions:
- Features: "feature/<description>"
- Bugfixes: "bugfix/<issuenumber>-<description>"
- Docs: "docs/<description>"

# PR conventions:
- Use the branch name (if possible)
- Link to the given issue (if it exists)
- Bugfix issues PR convention: "Fixed #1234, <short description>" and the number is a link to the issue.
- Label PRs with "Changelog: Feature" or "Changelog: Bugfix" as appropriate
